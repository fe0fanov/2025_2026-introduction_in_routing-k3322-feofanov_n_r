# yaml-конфиг

Конфиг во многом повторяет предыдущую лабораторную, только появился ещё один пк и немного другие связи. Каждый роутер подсоединён к своему компьютеру через eth3 и между собой через eth1 и eth2.

# Конфиги устройств

## Роутеры

В данной работе прописываются статические руты. DHCP аналогичен R01 из предыдущей работы.

В конфигах роутера пишем:

1. /ip address
2. /ip pool
3. /ip dhcp-server
4. /ip dhcp-server network
5. /ip route
6. /user (убираем admin и добавляем собственного пользователя)
7. /system identity

## Компьютеры

IP-адреса выдаются компьютерам через dhcp (`udhcpc -i eth1`). Но в Containerlab у них выставляется дефолтный маршрут от сети управления в yaml-конфиге, из-за чего попытки послать запрос на передаются на роутер моего провайдера, где этот запрос теряется. Поэтому необходимо удалить этот маршрут: `ip route del default via 10.20.30ю1 dev eth0`
